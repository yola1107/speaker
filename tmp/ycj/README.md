# 印钞机（YCJ）后端实现说明

游戏ID：`18926`

本说明仅描述 `game/ycj` 后端实现口径，不包含前端表现层（如 tips/动画/UI 文案）。

## 基础参数

- 盘面：`1 x 3`（`_rowCount=1`，`_colCount=3`）
- 基础投注系数：`20`（`_baseMultiplier=20`）
- 注额计算：`betAmount = BaseMoney * Multiple * 20`
- 主阶段：`base/free`

## 卷轴分工

| 卷轴位置 | 可出现符号 |
|----------|-----------|
| 第0列（左） | 空符号、数字符号（0.1/0.5/1/5/10） |
| 第1列（中） | 空符号、银行符号、翻倍符号、免费符号 |
| 第2列（右） | 空符号、数字符号（0.1/0.5/1/5/10） |

## 符号系统

- 左右列：空符号 + 数字符号（`0.1/0.5/1/5/10`）
- 中间列：空符号 + 银行 + 翻倍 + 免费符号
- 免费符号：`_free5/_free10/_free20`，对应加免 `5/10/20`

符号ID与类型见 `const.go`，符号倍数配置见 `game_json.go` 的 `symbol_mul`。

| ID范围 | 类型 | 说明 |
|--------|------|------|
| 0 | `_blank` | 空符号 |
| 1-5 | `_num01`-`_num10` | 数字符号（0.1/0.5/1/5/10） |
| 6 | `_bank` | 银行符号（中间符） |
| 7-14 | `_mul2`-`_mul100` | 翻倍符号（X2/X3/X5/X10/X20/X30/X50/X100） |
| 15-17 | `_free5`-`_free20` | 免费符号（5/10/20次） |

## 判奖与模式逻辑（后端实际）

### 中奖规则

在 `findWinInfos()` 中：

1. 中间为非空；
2. 左右均为数字；
3. 左右数字相同；

满足以上条件视为中奖。

### 倍数计算

在 `calculateMultiplier()` 中：

- `midVal = symbol_mul[mid]`
- 银行/免费符号的 `midVal` 视作 `1`
- 当左右相等时，最终倍数为：`leftVal * midVal`

> 注意：当前后端实现不是“左右两边都再乘一次并 /100”的口径。

### 推展模式（Extend）

触发条件：左右数字相同且中间为空。  
执行：中间列按当前滚轴向下掉一格（`getFallSymbol(1)`），再判奖。

### 重转模式（Respin）

触发条件：仅免费模式下，中间非空且左右数字不同。  
执行：左右列重新取符号（`getRandomSymbol(0/2)`），再判奖。

### 免费模式（夺宝）

触发条件：左右相等且中间为免费符号（5/10/20）。  
执行：正常判奖 + 增加免费次数。免费中可再次触发免费。

## RTP测试口径

`rtp_test.go` 与 `rtpx_test.go` 采用统一结构体 `benchmarkStats`：

- 统一累计字段：局数、触发次数、赔率收益、推展/重转统计等；
- 统一进度打印：`printBenchmarkProgress(stats, start)`；
- 统一汇总打印：`printBenchmarkSummary(stats, start)`。

### 统计口径说明

- 调试模式 `debug.open=true` 下，RTP测试以 `stepMultiplier` 作为收益统计基础；
- 报表中的“总投注(倍数)”按基础局扣费累加（每基础局 +20）；
- `rtpx_test.go` 额外输出逐步明细日志（用于复盘流程）。

## 策划口径 vs 后端口径（差异表）

| 项目 | 策划/旧文档口径 | 后端当前实现口径 |
|------|------------------|------------------|
| 中奖倍数公式 | 常见描述为左右数字与中间符共同参与（或有 `/100` 描述） | `calculateMultiplier()` 实际为 `leftVal * midVal`（左右相等时数字只乘一次） |
| 路数公式 | 常见模板有“第1列x第2列...”的多路计算 | 本游戏为 `1x3` 单次判奖，不使用多路计算模型 |
| 推展实现 | 常写“切换推展专用滚轴组” | 当前实现为中间列在当前滚轴 `getFallSymbol(1)` 下落一格 |
| 重转实现 | 常写“切换重转专用滚轴组” | 当前实现为免费模式下对左右列 `getRandomSymbol(0/2)` 重取符号 |
| RTP统计口径 | 多为金额口径理解 | 测试默认在 `debug.open=true` 下按 `stepMultiplier` 统计收益（非真实转账金额） |
| 中奖展示数据 | 可能单独依赖 `winGrid` | 当前后端判定与统计以 `cards`（当前盘面）+ `multiplier` 为主，`winGrid`不作为独立判奖来源 |

> 如需与策划金额口径完全一致，请在 RTP 测试中额外换算为 `betAmount * stepMultiplier` 再统计。

- `bet_order_helper.go`：判奖与倍数计算
- `bet_order_configs.go`：滚轴配置与取符
- `bet_order_scene.go`：场景状态持久化
- `rtp_test.go`：大样本 RTP 统计
