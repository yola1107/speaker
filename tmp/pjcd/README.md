# 《破茧成蝶》(Metamorphosis) 游戏设计文档

**项目名称:** 破茧成蝶 (PJCD)
**游戏类型:** 3×5 网格，20条赔付线，Cascading消除，多形态百搭
**开发语言:** Go
**参考项目:** Butterfly Blossom (PG Soft)

---

## 一、游戏概览

### 1.1 核心特色
- **黏性多形态百搭:** 毛虫 → 蝶茧 → 蝴蝶，参与中奖时形态升级
- **连击倍数系统:** 每轮中奖提升倍数，基础[1,2,3,5]，免费[3,6,9,15]
- **Cascading消除:** 中奖符号消除，上方符号下落补位，持续判定直到无中奖
- **免费游戏:** 触发条件3个及以上夺宝符号，免费模式中奖励大幅提升

### 1.2 基础配置
| 配置项 | 数值 | 说明 |
|--------|------|------|
| 网格大小 | 3×5 | 15个位置 |
| 赔付线 | 20条 | 固定线路 |
| 最大连击轮次 | 4轮 | 倍数递增至最高档 |

### 1.3 符号系统
| ID | 符号名称 | 3连 | 4连 | 5连 | 说明 |
|----|----------|-----|-----|-----|------|
| 1 | 三叶草 | 2x | 5x | 10x | 低值符号 |
| 2 | 长春花 | 2x | 5x | 10x | 低值符号 |
| 3 | 郁金香 | 5x | 8x | 20x | 中值符号 |
| 4 | 雏菊 | 5x | 8x | 20x | 中值符号 |
| 5 | 兰花 | 8x | 10x | 50x | 中高值符号 |
| 6 | 莲花 | 10x | 20x | 100x | 高值符号 |
| 7 | 玫瑰 | 20x | 50x | 200x | 最高值符号 |
| 8 | 百搭(WILD) | - | - | - | 替代其他符号 |
| 9 | 夺宝(SCATTER) | - | - | - | 触发免费游戏 |

## 二、游戏机制详解

### 2.1 基础模式玩法规则
```
用户点击SPIN → 扣除投注 → 生成盘面 → 判定中奖 → 中奖则应用轮次倍数 →
符号消除 → 上方符号下落 → 倍数提升 → 重复判奖 → 无中奖结算 →
统计夺宝符号数量 → 触发/不触发免费游戏
```

### 2.2 百搭形态机制
```
毛虫百搭(1) → (参与1次中奖) → 蝶茧百搭(2) → (再参与1次中奖) →
蝴蝶百搭(3) → (再参与1次中奖) → 消除并增加蝴蝶倍数
```

- 每轮中奖后，参与中奖的百搭形态升级
- 蝴蝶百搭参与中奖时，除正常赔付外，额外增加第4轮倍数
- 蝴蝶倍数在免费模式中累计，基础模式中每轮重置

### 2.3 连击倍数系统
- **基础模式:** [1, 2, 3, 5]
- **免费模式:** [3, 6, 9, 15] (基础倍数×3)
- 每次中奖后，倍数提升一档（不超过第4档）
- 无中奖时，连击结束，倍数重置

## 三、免费游戏机制

### 3.1 触发条件
| 夺宝符号数量 | 免费旋转次数 |
|--------------|--------------|
| 3个 | 8次 |
| 4个 | 10次 |
| 5个 | 12次 |

**计算公式:** 免费次数 = 8 + (夺宝数 - 3) × 2

### 3.2 免费模式内再触发
- **触发条件:** 2个及以上夺宝符号
- **新增次数:** 3 + (夺宝数 - 2) × 2

### 3.3 免费模式特殊设置
| 对比项 | 基础模式 | 免费模式 |
|--------|----------|----------|
| 符号权重 | 低值符号比例较高 | 高值符号比例提升 |
| WILD概率 | 2.75% | 5.90% (翻倍) |
| SCATTER概率 | 1.83% | 2.00% |
| 轮次倍数 | [1,2,3,5] | [3,6,9,15] |
| 蝴蝶倍数 | 每轮重置 | 累计到免费结束 |

## 四、轮轴构建机制

### 4.1 基础模式轮轴
**生成策略:** 每10次spin重新生成一套轮轴

**三步构建法:**
1. 按权重随机选择基础符号 (ID 1-7)
2. 按权重选择排列形式 (单符号81%, 二连17%, 三连2%)
3. 按概率替换为特殊符号 (SCATTER 1.83%, WILD 2.75%)

**WILD位置限制:** 仅中间三列(第2、3、4列)可出现WILD，首尾列不允许

### 4.2 免费模式轮轴
**生成策略:** 仅在触发免费游戏时生成一次，整个免费期间复用

**参数优化:**
- 高值符号权重提升 (玫瑰从9.25%→12%)
- WILD概率大幅增加 (2.75%→5.90%)
- SCATTER概率略有提升 (1.83%→2.00%)

## 六、关键算法流程

### 6.1 主流程
```
┌─────────────────────────────────────────────────────────────┐
│                      Spin 开始                              │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│        检查是否需要生成/刷新轮轴                            │
│  - 基础模式：每10次或首次，重新生成                        │
│  - 免费模式：免费开始时生成，期间复用                       │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                   生成当前盘面                              │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    执行判奖流程                              │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    有中奖？                                 │
│    是 → 应用倍数 → 累加中奖额 → 进入消除                    │
│    否 → 检查夺宝 → 触发免费游戏/结束                       │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                 符号消除 → 上方下落                         │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│               百搭形态升级 & 蝴蝶倍数处理                    │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                   倍数提升 & 继续判奖                        │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 百搭处理逻辑
- 每次中奖后，参与中奖的百搭符号形态升级
- 同一消除步中，多个中奖可能涉及同一百搭，只升级一次
- 蝴蝶百搭参与中奖时，除正常赔付外，额外增加蝴蝶倍数

## 八、性能优化

### 8.1 轮轴缓存
- 基础模式轮轴每10次生成一次，复用期间避免重复计算
- 免费模式轮轴整个期间复用，减少生成开销

### 8.2 内存管理
- 复用数据结构，避免频繁分配
- 及时释放不必要的中间数据
