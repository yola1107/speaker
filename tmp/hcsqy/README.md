## 核心玩法

### 1. 百搭变大

**触发条件：**
- 符号下落完成后，中奖判定前
- 某列有 1-2 个百搭，且该列无夺宝

**效果：**
- 播放百搭变大的动画（前端表现）
- 将被选中的那一列，将原本的百搭（以及该列上与百搭相关的表现位）替换/升级为「长条百搭」，并让长条百搭**占据整列（3格）**
- 多列符合条件时：在所有候选列中**随机选 1 列**执行变大
- 触发概率：满足“候选列存在”后，再进行一次 5% 判定；因此在统计意义上实际触发概率会低于 5%

**影响范围说明：**
- 只有被选中的那一列会变大；其它列不变。
- 若该列存在夺宝（黑卡夺宝，ID=9），则该列不属于候选列，不会触发变大。
- 长条百搭携带倍数（x2 - x20），该倍数用于后续“线奖”奖励计算。

**注意：** 重转至赢模式中不触发百搭变大

### 2. 长条百搭倍数

长条百搭携带倍数（x2 - x20），线中奖乘以倍数为最终奖励。

| 倍数 | 2 | 3 | 4 | 5 | 6 | 8 | 10 | 12 | 15 | 18 | 20 |
|------|---|---|---|---|---|---|----|----|----|----|----|
| 概率 | 0.23 | 0.2 | 0.16 | 0.12 | 0.1 | 0.08 | 0.05 | 0.03 | 0.015 | 0.01 | 0.005 |

### 2.1 图例说明（普通百搭 vs 长条百搭）

变化前（普通百搭）：            变化后（长条百搭）：

```
┌────┬────┬────┐            ┌────┬────┬────┐
│ 8  │    │    │            │ 8  │    │    │  ← 携带倍数（如 x5）
├────┼────┼────┤            ├────┼────┼────┤
│    │  5 │    │     →      │ 8  │  5 │    │  ← 整列变成长条百搭
├────┼────┼────┤            ├────┼────┼────┤
│ 8  │    │  3 │            │ 8  │    │  3 │
└────┴────┴────┘            └────┴────┴────┘
```

示例：第 0 列有 2 个百搭 → 触发变大后，第 0 列**整列变为长条百搭**（3格全是百搭），且携带同一个倍数（例如 x5）。

### 2.2 长条百搭的来源

```
┌──────────┬──────────────────────────────────┬────────────┐
│   来源   │             触发场景             │ 是否带倍数 │
├──────────┼──────────────────────────────────┼────────────┤
│ 百搭变大 │ 基础/免费：先候选列，再 5%，再随机列 │ ✅ x2-x20  │
├──────────┼──────────────────────────────────┼────────────┤
│ 重转至赢 │ 重转至赢每次重转，随机一列变为   │ ✅ x2-x20  │
│          │ 长条百搭                         │            │
└──────────┴──────────────────────────────────┴────────────┘
```

### 3. 重转至赢

**触发：** 下注后概率触发，滚动停止前判定

**机制：**
- 进入后若无中奖，重转直到中奖
- 每次重转，随机一列变为长条百搭
- 不保留口径：上一轮重转生成的长条百搭不会延续到下一次重转；因此任意单次盘面中长条百搭列数最多为 1 列
- 重转至赢中生成的”长条百搭”来自重转规则本身；
  不再执行普通/免费里的”百搭变大（候选列 + 概率判定 + 升级为长条百搭）”逻辑
- 有炸胡动画

**回包字段（联调）：重转至赢 vs 长条列**

| 字段 | 含义 |
|------|------|
| `next` | 为 `true` 时：本手尚未结束，客户端需 **再请求一次** spin（重转至赢、未出奖时）。 |
| `isRespinUntilWin` | 为 `true` 时：服务端仍处在重转至赢链上，**下一请求**会继续走重转（与 `next` 同时为真表示还要继续打）。出奖结束重转后为 `false`。 |
| `respinWildCol` | **本包**重转至赢流程中随机到的长条百搭列 **0～2**；**非重转模式时为 -1**。出奖结束重转的 **当包** 仍会带回 **当次** 盖百搭的列号，便于播炸胡/列光效。 |
| `wildMultiplier` | 本步长条百搭倍数（重转至赢与百搭变大共用该字段）。 |
| `lineMultiplier` | 本步各中奖线 **基础赔率之和**（未乘 `wildMultiplier`）；与 `winInfo.freeGameMultiple` 一致满足：`freeGameMultiple = lineMultiplier × wildMultiplier`。 |

### 4. 免费游戏

**触发条件：**
- 3 个夺宝 = 10 次免费
- 每多 1 个夺宝 = +2 次免费

**免费中：**
- 夺宝符号可触发免费中免费；新增免费次数计算口径与触发免费一致：当本轮夺宝符号数 `S >= 3` 时，新增免费 = `10 + 2*(S-3)`。
- 其中“夺宝2+1”仅为前端动画特效，服务端不额外增加免费次数（即不会因 `S=2` 而按 2+1 机制补免费）。
- 百搭变大和重转至赢正常触发

### 5. 购买免费

- 价格：75 倍 bet
- 直接进入免费模式

## 6. 中奖计算（口径描述）

### 6.1 投注额

建议以策划口径统一（本 README 待你确认最终实现）：

- `下注额 = 投注大小 x 投注倍数 x 基础倍率(5)`

其中：
- `投注大小`：0.04 | 0.40 | 4.00
- `投注倍数`：1 - 10
- 基础倍率：5（基础投注设定）

### 6.2 单条中奖奖励

根据你的描述，线奖计算结构为：

- `单条线中奖奖励 = 下注额 x 符号赔付值 x 倍数`

倍数来源：
- 百搭变大后的长条百搭倍数（x2 - x20）
- 重转至赢模式内触发的长条百搭倍数（同样携带 x2 - x20）

> 注：如果你们实现里”倍数还会乘以额外系数（例如重转至赢/长条百搭的二次乘区分）”，请把最终公式补充进来，本 README 保持为待补齐口径。

## 盘面坐标

3x3 盘面，索引从左上角 0 开始，到右下角 8 结束：

```
     列0   列1   列2
    ┌─────┬─────┬─────┐
行0 │  0  │  1  │  2  │
    ├─────┼─────┼─────┤
行1 │  3  │  4  │  5  │
    ├─────┼─────┼─────┤
行2 │  6  │  7  │  8  │
    └─────┴─────┴─────┘
```

索引与行列坐标对应关系：`索引 = 行 * 3 + 列`

## 中奖线定义

5条线：横线3条 + 对角线2条

```
行0横线: 索引 [0, 1, 2]  → (0,0) - (0,1) - (0,2)
行1横线: 索引 [3, 4, 5]  → (1,0) - (1,1) - (1,2)
行2横线: 索引 [6, 7, 8]  → (2,0) - (2,1) - (2,2)
对角线1: 索引 [0, 4, 8]  → (0,0) - (1,1) - (2,2)  // 左上-右下
对角线2: 索引 [6, 4, 2]  → (2,0) - (1,1) - (0,2)  // 左下-右上
```

**lines 数组（二维坐标）：**

```go
var lines = [][]int{
    {0, 1, 2}, // 行0横线
    {3, 4, 5}, // 行1横线
    {6, 7, 8}, // 行2横线
    {0, 4, 8}, // 对角线1（左上-右下）
    {6, 4, 2}, // 对角线2（左下-右上）
}
```
